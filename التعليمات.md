# خطوات تفعيل التطبيق (Firebase) — بالتفصيل

## 1) الإعداد ✅ (تم بالفعل)
مفاتيح Firebase الخاصة بمشروعك (`courstrack`) والرمز السري للأستاذ (`chaker1234`) مضبوطون مسبقًا في `index.html` — لا حاجة لأي تعديل في هذه النقاط.

## 2) قاعدة بيانات Firestore
إن لم تكن أنشأتها بعد: من Firebase Console → Build → Firestore Database → Create database → اختر "Start in production mode" وأقرب موقع خادم.

## 3) قواعد الأمان (Security Rules) — الخطوة المطلوبة منك الآن
من Firestore Database → Rules، الصق هذا بالضبط ثم اضغط "Publish":

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /students/{studentId} {
      allow read: if true;
      allow create, update: if request.resource.data.teacherKey == "chaker1234";
      allow delete: if resource.data.teacherKey == "chaker1234";
    }
    match /sessions/{sessionId} {
      allow read: if true;
      allow create, update: if request.resource.data.teacherKey == "chaker1234";
      allow delete: if resource.data.teacherKey == "chaker1234";
    }
  }
}
```
اضغط "Publish".

⚠️ **ملاحظة صراحة عن الأمان**: هذا الأسلوب (رمز سري داخل الكود) مناسب لاستعمال شخصي بسيط (أستاذ واحد، بيانات دروس خصوصية غير حسّاسة)، لكنه ليس تشفيرًا حقيقيًا — أي شخص يفحص كود الصفحة يقدر يرى الرمز. لا تستعمله لبيانات حسّاسة (كأرقام بطاقات أو معلومات طبية). إذا احتجت أمانًا أقوى مستقبلًا (تسجيل دخول حقيقي بالبريد وكلمة مرور)، أخبرني وأضيفه.

## 4) الرفع على GitHub Pages
نفس الطريقة المعتادة: ارفع كل الملفات (index.html, manifest.json, sw.js, icon-192.png, icon-512.png) في جذر الـ repo، وفعّل Pages من Settings.

## 5) التجربة
- افتح الرابط → "بوابة الأستاذ" → أدخل الرمز السري → أضف تلميذًا → افتح ملفه → انسخ "رابط ولي الأمر".
- أرسل هذا الرابط لولي الأمر (واتساب مثلاً) — سيفتح مباشرة على سجلّ ابنه بدون الحاجة لأي دخول أو رمز.

## حدود يجب معرفتها
- الخطة المجانية من Firebase (Spark) تكفي بسهولة لاستعمال شخصي (حتى آلاف القراءات/الكتابات يوميًا مجانًا).
- لا يوجد نسخ احتياطي تلقائي مرئي لك — البيانات تبقى في Firebase طالما لم تحذف المشروع.
