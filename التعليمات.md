# خطوات تفعيل التطبيق (Firebase) — بالتفصيل

## 1) إنشاء مشروع Firebase (مجاني)
1. افتح https://console.firebase.google.com وسجّل الدخول بحساب Google.
2. اضغط "Add project" → أعطه اسمًا (مثلاً: متابعة-الدروس) → أكمل الخطوات (يمكنك تعطيل Google Analytics، غير ضروري).

## 2) إنشاء قاعدة بيانات Firestore
1. من القائمة الجانبية: Build → Firestore Database → Create database.
2. اختر "Start in production mode" ثم اختر أقرب موقع خادم (مثلاً: eur3 لأوروبا).

## 3) الحصول على مفاتيح الربط (Config)
1. من الصفحة الرئيسية للمشروع، اضغط أيقونة الويب `</>` لإضافة تطبيق ويب.
2. أعطه اسمًا واضغط "Register app".
3. سيظهر لك كائن `firebaseConfig` يحتوي على: apiKey, authDomain, projectId, storageBucket, messagingSenderId, appId.
4. افتح ملف `index.html` وابحث عن `PASTE_YOUR_API_KEY_HERE` واستبدل كل القيم بما ظهر لك.

## 4) تعيين رمز سري للأستاذ (PIN)
في نفس الملف `index.html`، ابحث عن هذا السطر:
```
const TEACHER_PIN = "CHANGE_ME_1234";
```
غيّره إلى رمز من اختيارك (احفظه جيدًا، هو مفتاح دخولك لبوابة الأستاذ).

## 5) قواعد الأمان (Security Rules) — مهم جدًا
من Firestore Database → Rules، الصق هذا (وغيّر `CHANGE_ME_1234` لنفس الرمز اللي حطيته في الخطوة 4):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /students/{studentId} {
      allow read: if true;
      allow create, update: if request.resource.data.teacherKey == "CHANGE_ME_1234";
      allow delete: if resource.data.teacherKey == "CHANGE_ME_1234";
    }
    match /sessions/{sessionId} {
      allow read: if true;
      allow create, update: if request.resource.data.teacherKey == "CHANGE_ME_1234";
      allow delete: if resource.data.teacherKey == "CHANGE_ME_1234";
    }
  }
}
```
اضغط "Publish".

⚠️ **ملاحظة صراحة عن الأمان**: هذا الأسلوب (رمز سري داخل الكود) مناسب لاستعمال شخصي بسيط (أستاذ واحد، بيانات دروس خصوصية غير حسّاسة)، لكنه ليس تشفيرًا حقيقيًا — أي شخص يفحص كود الصفحة يقدر يرى الرمز. لا تستعمله لبيانات حسّاسة (كأرقام بطاقات أو معلومات طبية). إذا احتجت أمانًا أقوى مستقبلًا (تسجيل دخول حقيقي بالبريد وكلمة مرور)، أخبرني وأضيفه.

## 6) الرفع على GitHub Pages
نفس الطريقة المعتادة: ارفع كل الملفات (index.html, manifest.json, sw.js, icon-192.png, icon-512.png) في جذر الـ repo، وفعّل Pages من Settings.

## 7) التجربة
- افتح الرابط → "بوابة الأستاذ" → أدخل الرمز السري → أضف تلميذًا → افتح ملفه → انسخ "رابط ولي الأمر".
- أرسل هذا الرابط لولي الأمر (واتساب مثلاً) — سيفتح مباشرة على سجلّ ابنه بدون الحاجة لأي دخول أو رمز.

## حدود يجب معرفتها
- الخطة المجانية من Firebase (Spark) تكفي بسهولة لاستعمال شخصي (حتى آلاف القراءات/الكتابات يوميًا مجانًا).
- لا يوجد نسخ احتياطي تلقائي مرئي لك — البيانات تبقى في Firebase طالما لم تحذف المشروع.
